## >> WHAT IS ALGOARENA?

AlgoArena is a full-stack exam platform where admins can generate AI-powered quizzes from any syllabus or topic, publish them as live sessions, and monitor candidates in real time вҖ” all from a single dashboard.

Candidates join with a session ID and password, attempt the quiz within a time limit, and get scored instantly. Admins can review every answer, override scores manually, and detect suspicious behavior via tab-switch tracking.

```
ADMIN                          CANDIDATE
  в”Ӯ                               в”Ӯ
  в”ңв”Җ Generate quiz (AI)           в”ңв”Җ Sign up / Login
  в”ңв”Җ Edit questions               в”ңв”Җ Join session (ID + password)
  в”ңв”Җ Set timer & expiry           в”ңв”Җ Attempt quiz
  в”ңв”Җ Publish вҶ’ Supabase           в”ңв”Җ Auto-submit on timeout
  в”ңв”Җ Watch dashboard live         в””в”Җ Get score instantly
  в”ңв”Җ View candidate logs
  в”ңв”Җ Override scores
  в””в”Җ Terminate session
```

### Data Flow вҖ” Quiz Generation

```
Admin fills form (title, syllabus, count, timer, password)
    в”Ӯ
    в–ј
POST /generate вҶ’ api/generate.js
    в”Ӯ
    в–ј
Groq LLaMA 3.1 8B вҶ’ JSON quiz object
    в”Ӯ
    в–ј
Normalize + pad to exact question count
    в”Ӯ
    в–ј
Return to Angular вҶ’ examgen editor
    в”Ӯ
    в–ј
Admin edits/previews вҶ’ clicks Publish
    в”Ӯ
    в–ј
POST /create-session вҶ’ api/create-session.js
    в”Ӯ
    в”ңв”Җв”Җ Verify admin token (Supabase Auth)
    в”ңв”Җв”Җ Check role === 'admin' in profiles table
    в”ңв”Җв”Җ gzip compress quiz data вҶ’ base64
    в””в”Җв”Җ INSERT into sessions table (Supabase)
```

### Data Flow вҖ” Candidate Attempt

```
Candidate enters session ID + name + password
    в”Ӯ
    в–ј
POST /join-session вҶ’ api/join-session.js
    в”Ӯ
    в”ңв”Җв”Җ Verify candidate token
    в”ңв”Җв”Җ Check session exists + not expired + not closed
    в”ңв”Җв”Җ Check password matches
    в”ңв”Җв”Җ Check no duplicate submission
    в”ңв”Җв”Җ Decompress (base64 вҶ’ gunzip вҶ’ JSON)
    в””в”Җв”Җ Return questions + timer
    в”Ӯ
    в–ј
Candidate attempts quiz (timer counts down)
    в”Ӯ
    в”ңв”Җв”Җ Tab switch detected вҶ’ tabSwitches++
    в””в”Җв”Җ Timer hits 0 вҶ’ auto-submit
    в”Ӯ
    в–ј
POST /submit-exam вҶ’ api/submit-exam.js
    в”Ӯ
    в”ңв”Җв”Җ Verify candidate token
    в”ңв”Җв”Җ Decompress questions from session
    в”ңв”Җв”Җ Grade answers (case-insensitive match)
    в”ңв”Җв”Җ Calculate suspicion (CLEAN/MEDIUM/HIGH)
    в””в”Җв”Җ INSERT into submissions table
```

## >> DATABASE SCHEMA

```sql
-- User profiles (extends Supabase Auth)
profiles
  в”ңв”Җв”Җ id          uuid  PK вҶ’ auth.users
  в”ңв”Җв”Җ email       text
  в”ңв”Җв”Җ full_name   text
  в”ңв”Җв”Җ phone       text
  в”ңв”Җв”Җ role        text  CHECK ('admin' | 'candidate')
  в””в”Җв”Җ created_at  timestamptz

-- Exam sessions
sessions
  в”ңв”Җв”Җ id              text  PK  (format: ALGO-XXXX)
  в”ңв”Җв”Җ owner_id        uuid  вҶ’ auth.users
  в”ңв”Җв”Җ title           text
  в”ңв”Җв”Җ description     text
  в”ңв”Җв”Җ password        text
  в”ңв”Җв”Җ validity_start  timestamptz
  в”ңв”Җв”Җ expiry          timestamptz
  в”ңв”Җв”Җ submit_timeout  int   (minutes)
  в”ңв”Җв”Җ max_candidates  int
  в”ңв”Җв”Җ data            text  (gzip compressed + base64 encoded quiz)
  в”ңв”Җв”Җ closed          boolean
  в””в”Җв”Җ created_at      timestamptz

-- Candidate submissions
submissions
  в”ңв”Җв”Җ id            uuid  PK
  в”ңв”Җв”Җ session_id    text  вҶ’ sessions
  в”ңв”Җв”Җ candidate_id  uuid  вҶ’ auth.users
  в”ңв”Җв”Җ answers       jsonb
  в”ңв”Җв”Җ score         int
  в”ңв”Җв”Җ status        text
  в”ңв”Җв”Җ tab_switches  int
  в”ңв”Җв”Җ suspicion     text  ('CLEAN' | 'MEDIUM' | 'HIGH')
  в”ңв”Җв”Җ time_taken    int   (seconds)
  в””в”Җв”Җ submitted_at  timestamptz
```

**Auto-cleanup:** pg_cron deletes expired sessions every 10 minutes.

## >> API REFERENCE

All endpoints are Vercel Serverless Functions. In development they're proxied via `proxy.conf.json` to `localhost:3000`.

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `POST` | `/generate` | вҖ” | Generate quiz via Groq AI |
| `POST` | `/create-session` | Admin token | Publish quiz to Supabase |
| `GET` | `/get-sessions` | Admin token | Fetch admin's active sessions |
| `GET` | `/get-submissions` | Admin token | Fetch session's candidate submissions |
| `POST` | `/join-session` | Candidate token | Validate + join a session |
| `POST` | `/submit-exam` | Candidate token | Grade + save answers |
| `POST` | `/update-score` | Admin token | Override candidate score |
| `POST` | `/terminate-session` | Admin token | Delete session |

### POST /generate
```json
// Request
{ "syllabus": "Binary Search Trees", "count": 10 }

// Response
{ "questions": [{ "type": "MCQ", "text": "...", "options": [...], "correctAnswer": "..." }] }
```

### POST /create-session
```json
// Request
{
  "title": "DSA Mid-Term",
  "description": "...",
  "questions": [...],
  "password": "secret123",
  "validityStart": "2026-05-24T10:00:00Z",
  "expiryDateTime": "2026-05-24T12:00:00Z",
  "submitTimeout": 60,
  "maxCandidates": 50,
  "ownerToken": "eyJ..."
}

// Response
{ "success": true, "sessionId": "ALGO-4821" }
```

## >> GETTING STARTED

### Prerequisites

- Node.js 18+
- Angular CLI 21
- Supabase account
- Groq API key

### Local Development

```bash
# 1. Clone the repo
git clone https://github.com/Bhavish089/AlgoArena.git
cd AlgoArena

# 2. Install dependencies
npm install

# 3. Create .env file
cp .env.example .env
# Fill in: GROQ_API_KEY, SUPABASE_URL, SUPABASE_SERVICE_KEY

# 4. Start the backend server (terminal 1)
npm run server

# 5. Start the Angular dev server (terminal 2)
npm run dev
# Runs on http://localhost:4200
# API calls proxy to localhost:3000 via proxy.conf.json
```

### Environment Variables

| Variable | Where | Description |
|---|---|---|
| `GROQ_API_KEY` | `.env` + Vercel | Groq API key for AI generation |
| `SUPABASE_URL` | `.env` + Vercel | Your Supabase project URL |
| `SUPABASE_SERVICE_KEY` | `.env` + Vercel | Supabase service role key (server-side only) |

### Supabase Setup

1. Create a new Supabase project
2. Run the schema SQL (see `database/schema.sql` or the Database Schema section above)
3. Enable `pg_cron` extension for auto-cleanup
4. Add your Supabase URL + publishable key to `src/environments/environment.ts`

### Deploy to Vercel

```bash
# Push to GitHub вҖ” Vercel auto-deploys on every push to main
git push origin main
```

Add these environment variables in **Vercel вҶ’ Settings вҶ’ Environment Variables**:
- `GROQ_API_KEY`
- `SUPABASE_URL`
- `SUPABASE_SERVICE_KEY`

## >> FEATURES

### For Admins
- **AI Quiz Generation** вҖ” Enter any syllabus topic, get MCQ/TF/Short answer questions instantly
- **Question Editor** вҖ” Edit, add, remove questions before publishing
- **Preview Mode** вҖ” Review the full quiz before going live
- **Session Control** вҖ” Set validity window, time limit, password, max candidates
- **Live Dashboard** вҖ” See candidates as they submit in real time (polls every 10s)
- **Detailed Logs** вҖ” View every answer per candidate with correct/incorrect breakdown
- **Score Override** вҖ” Manually adjust any candidate's score
- **Terminate** вҖ” End a session instantly

### For Candidates
- **Simple Join** вҖ” Session ID + name + password
- **Live Timer** вҖ” Countdown with auto-submit on expiry
- **MCQ / True-False / Short Answer** вҖ” All question types supported
- **Instant Score** вҖ” See result immediately after submission
- **Anti-cheat** вҖ” Tab switches tracked and flagged (CLEAN / MEDIUM / HIGH)

### Platform
- **3 Themes** вҖ” Light, Dark, High-Contrast
- **Fully Responsive** вҖ” Mobile, tablet, desktop
- **Role-based** вҖ” Admin and candidate accounts
- **Secure** вҖ” Supabase RLS, token-based auth on all endpoints
- **Compressed Storage** вҖ” Quiz data gzip compressed before saving to DB
